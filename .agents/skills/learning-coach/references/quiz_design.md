# 出题设计规则

## 出题来源优先级

默认顺序：
1. 当前材料中的重点
2. 理解重点所需的必要补充
3. 围绕重点的经典题型或网络延伸

第三类只能少量加入，用来增强迁移能力或面试感，不应喧宾夺主。

## 比例建议

默认比例：
- 材料重点题：70% 到 100%
- 经典题与网络延伸题：0% 到 30%

如果材料本身已经很完整，优先接近 100% 材料题。
如果用户明确说“想要更强的迁移和应用”，再适当加入延伸题。

## 经典题与网络延伸题的限制

经典题或网络延伸题只有在满足下面条件时才可使用：
- 它和当前重点直接相关
- 它不是只和某段补充背景相关
- 它不会把当前场景从“学重点”带偏成“学枝节”

错误做法：
- 材料里顺带提过一个背景概念，就围绕那个背景概念出很多题
- 因为网络搜索到了很多资料，就把题目中心切到搜索结果而不是当前材料

## 场景感知

不同场景的题型应不同：

- 先搭框架：适合概念关系题、分类题、主线辨析题
- 精读材料：适合细节理解题、因果题、定位题
- 面试表达：适合结构化问答题、追问题
- 专项补弱：适合针对某个误区反复变式

## JSON 输出的意义

题目脚本除了 Markdown，还应输出 JSON。

原因：
- Markdown 方便当前直接阅读和作答
- JSON 方便后续接插件或 app，渲染成可点击的选择题面板
- 同一份题目逻辑可以被不同界面复用
